# Engineering Standards

## Overview

A centralized set of standards applied to all code generations within a team or repository.

## When to Use This Skill

- Initializing a new module or component.
- Reviewing code for adherence to team practices.
- Deciding on architectural patterns for new features.

## Core Standards

1. **Style**: Use consistent naming (e.g., camelCase for JS).
2. **Security**: Defense-in-depth validation for all user inputs.
3. **Scalability**: Design for future state (e.g., 10x current data volume).
4. **Debugging**: Use full code path tracing instead of just fixing symptoms.

## Anti-Patterns

❌ **Ad-hoc Patterns**: Introducing new patterns without documenting them here.
❌ **Symptom-Fixing**: Fixing a bug without identifying its root cause in the stack.
