# Frontend Design

## Overview

This skill guides creation of distinctive, production-grade frontend interfaces that avoid generic "AI slop" aesthetics. Implement real working code with exceptional attention to aesthetic details and creative choices.

## When to Use This Skill

- Building web components, pages, or applications.
- When high visual quality and unique design are required.
- To avoid "AI-looking" generic frontend designs.

## Core Principles

1. **Bold Aesthetic Direction**: Commit to a specific theme (e.g., brutally minimal, maximalist, retro-futuristic) before coding.
2. **Typography Excellence**: Use characterful fonts instead of generic ones like Inter or Arial.
3. **Intentional Motion**: Use CSS animations and micro-interactions to create delight.
4. **Spatial Innovation**: Use unexpected layouts, asymmetry, and grid-breaking elements.

## Step-by-Step Process

### Phase 1: Planning

**Actions:**

- Understand purpose and audience.
- Select a bold aesthetic theme.
- Identify one "unforgettable" feature.

### Phase 2: Execution

**Actions:**

- Implement with distinctive typography and curated colors.
- Add background textures or effects (e.g., noise, mesh gradients).
- Orchestrate staggered entry animations.

## Anti-Patterns

❌ **AI Slop**: Overused fonts (Inter), purple-on-white gradients, generic dashboard layouts.
❌ **Timid Palettes**: Evenly distributed, safe colors that lack character.
